## 一、项目概述

### 1.1 项目定位

本项目是一个工业电机预测性维护系统的**Demo原型**，用于在低成本硬件上验证核心闭环：

```
传感器采集 → RTOS任务调度 → 特征提取 → 健康指数计算 → 故障等级判断 → 本地报警 → 数据上报
```

### 1.2 与正式产品的关系

本Demo运行在STM32F103C8T6上，是后续迁移到STM32H750/H743工业级平台前的功能原型。设计原则是：

- **保持软件架构一致**：任务划分、数据结构、诊断模型在H7上可直接复用
- **降低硬件门槛**：用现有硬件替代工业级传感器，算法思想不变
- **功能可扩展**：预留OLED显示、W25Q64存储等接口，后续Phase可启用

### 1.3 核心功能清单

| 序号 | 功能 | 说明 |
|------|------|------|
| 1 | 振动采集 | MPU6050三轴加速度，100Hz~500Hz |
| 2 | 温度采集 | NTC热敏电阻，ADC读取 |
| 3 | 电流模拟 | 电位器模拟电流传感器输入 |
| 4 | 转速测量 | 反射式红外传感器检测风扇叶片脉冲 |
| 5 | 电机控制 | TB6612驱动130直流电机，PWM调速+制动 |
| 6 | 特征提取 | RMS、峰值、峰峰值、方差 |
| 7 | 健康诊断 | 健康指数0~100，四级故障等级 |
| 8 | 本地报警 | LED三色指示+蜂鸣器+电机制动 |
| 9 | 串口上报 | JSON格式数据输出到PC |
| 10 | PC上位机 | Python实时曲线显示+CSV记录 |

### 4.2 各模块接线详解

#### 4.2.1 MPU6050振动传感器（I2C）

MPU6050模块有8个引脚，我们使用其中4个：

```
MPU6050模块              STM32F103
┌──────────┐            ┌──────────┐
│ VCC  ────┼────────────┤ 3.3V     │
│ GND  ────┼────────────┤ GND      │
│ SCL  ────┼────────────┤ PB6 (I2C1_SCL)
│ SDA  ────┼────────────┤ PB7 (I2C1_SDA)
│ AD0  ────┼────────────┤ GND      │  ← 地址=0x68
│ INT  ────┤ (不接)     │          │
│ XDA  ────┤ (不接)     │          │
│ XCL  ────┤ (不接)     │          │
└──────────┘            └──────────┘
```

**注意事项：**
- AD0接GND设置I2C地址为0x68；接VCC则为0x69
- 模块上通常已有I2C上拉电阻，无需外接
- MPU6050需要贴在130电机外壳上才能感应振动——用双面胶或扎带固定
- 振动信号很微弱，贴得越紧检测效果越好

#### 4.2.2 OLED显示屏（I2C，与MPU6050共享总线）

OLED模块有4个引脚：

```
OLED 0.96寸             STM32F103
┌──────────┐            ┌──────────┐
│ VCC  ────┼────────────┤ 3.3V     │
│ GND  ────┼────────────┤ GND      │
│ SCL  ────┼──────┬─────┤ PB6 (I2C1_SCL)  ← 与MPU6050共享
│ SDA  ────┼──────┼┬────┤ PB7 (I2C1_SDA)  ← 与MPU6050共享
└──────────┘      ││    └──────────┘
                  ││
MPU6050的SCL/SDA也接在这两条线上（并联）
```

**注意事项：**
- OLED地址通常是0x3C（SSD1306驱动），与MPU6050(0x68)不冲突
- 使用前需先焊接排针到OLED模块的焊盘上
- I2C总线共享时，软件中需用Mutex_I2C互斥量保护访问

#### 4.2.6 TB6612电机驱动 + 130电机

```
TB6612模块              STM32F103              130电机
┌──────────┐           ┌──────────┐           ┌──────────┐
│ VM   ────┼───────────┤ 5V (面包板电源)       │          │
│ VCC  ────┼───────────┤ 3.3V     │           │  M+  ────┤←─ AO1
│ GND  ────┼───────────┤ GND      │           │  M-  ────┤←─ AO2
│ PWMA ────┼───────────┤ PA0 (TIM2_CH1)       └──────────┘
│ AIN2 ────┼───────────┤ PB9 (GPIO)
│ AIN1 ────┼───────────┤ PB8 (GPIO)
│ STBY ────┼───────────┤ 3.3V     │  ← 常使能
│ AO1  ────┼───────→ 电机端子1 (红/蓝线之一)
│ AO2  ────┼───────→ 电机端子2 (另一根线)
│ BIN1,BIN2,PWMB,BO1,BO2 ─── 不接（通道B不用）
└──────────┘
```

**电机控制真值表：**

| AIN1(PB8) | AIN2(PB9) | PWMA(PA0) | 电机状态 |
|-----------|-----------|-----------|---------|
| 0 | 0 | X | 滑行（惯性停转） |
| 0 | 1 | PWM | 正转 |
| 1 | 0 | PWM | 反转 |
| 1 | 1 | X | **制动刹车**（紧急停机用） |

**注意事项：**
- VM接5V（电机动力电源），VCC接3.3V（逻辑电源）
- STBY接3.3V让模块始终处于工作状态
- 130电机两根线无极性，交换AO1/AO2只是改变转向
- 风扇叶安装在电机轴上，用螺丝固定

#### 4.2.7 反射式红外传感器（测速）

```
红外传感器模块           STM32F103
┌──────────┐            ┌──────────┐
│ VCC  ────┼────────────┤ 3.3V     │
│ GND  ────┼────────────┤ GND      │
│ AO   ────┤ (不接)     │          │
│ DO   ────┼────────────┤ PB5 (EXTI5)  ← 数字脉冲输出
└──────────┘            └──────────┘
```

**安装方法：**
1. 将红外传感器模块固定在面包板上
2. 调整传感器方向使其正对风扇叶片侧面
3. 距离约5~15mm
4. 转动模块上的电位器，调节灵敏度直到风扇转动时DO指示灯能随叶片闪动

**测速原理：**
- 风扇有3片叶片
- 每片叶片经过时触发一次反射检测
- STM32通过EXTI中断计数脉冲
- 每500ms统计一次脉冲数，换算RPM
- RPM = 脉冲数 × 40（3叶片 × 2次/秒采样 = RPM = pulses × 60 / 3 / 2）

#### 4.2.8 LED指示灯（三色）

```
STM32F103           1KΩ电阻        LED          GND
PA8  ──────────────┤1KΩ├──────────┤ +  │──────── GND    (绿色 - 正常)
PA11 ──────────────┤1KΩ├──────────┤ +  │──────── GND    (黄色 - 预警)
PA12 ──────────────┤1KΩ├──────────┤ +  │──────── GND    (红色 - 报警)
```

**接线方式：** GPIO引脚 → 1KΩ电阻 → LED长脚(正极) → LED短脚(负极) → GND

**LED亮度：** 3.3V / 1KΩ ≈ 3mA，足够亮且不会损坏GPIO。

#### 4.2.9 有源蜂鸣器（低电平触发）

```
蜂鸣器模块            STM32F103
┌──────────┐           ┌──────────┐
│ VCC  ────┼───────────┤ 3.3V     │
│ GND  ────┼───────────┤ GND      │
│ I/O  ────┼───────────┤ PB3 (GPIO)    ← 输出LOW响，HIGH不响
│ (第4脚)──┤ 不接或GND  │          │
└──────────┘           └──────────┘
```

#### 4.2.10 CH340 USB转TTL（串口调试/上报）

```
CH340模块              STM32F103
┌──────────┐           ┌──────────┐
│ VCC  ────┤ 不接      │          │  ← CH340由PC USB供电
│ GND  ────┼───────────┤ GND      │  ← 必须共地!
│ TXD  ────┼───────────┤ PA10 (USART1_RX)  ← CH340的TX接STM32的RX
│ RXD  ────┼───────────┤ PA9  (USART1_TX)  ← CH340的RX接STM32的TX
└──────────┘           └──────────┘
```

**注意：** TX接RX，RX接TX，交叉连接！CH340模块由PC的USB口供电，VCC可不接。

#### 4.2.11 ST-LINK下载器

```
ST-LINK                STM32F103
┌──────────┐           ┌──────────┐
│ SWDIO ───┼───────────┤ PA13     │
│ SWCLK ───┼───────────┤ PA14     │
│ GND   ───┼───────────┤ GND      │
│ 3.3V  ───┤ 不接或接   │ 3.3V     │  ← 若STM32已USB供电则不接
└──────────┘           └──────────┘
```

### 4.3 电源方案

```
                    ┌──────────────────┐
                    │   面包板电源模块   │
PC USB ──→ STM32   │  输入：6.5~12V DC │
(供电)              │  输出：5V / 3.3V  │
                    └────────┬─────────┘
                             │
               ┌─────────────┼─────────────┐
               │             │             │
             3.3V轨         5V轨          GND轨
          ┌────┴────┐    ┌───┴───┐    ┌────┴────┐
          │MPU6050  │    │TB6612  │    │ 所有模块│
          │OLED     │    │ VM     │    │ 的GND  │
          │W25Q64   │    │红外传感│    │ 连一起 │
          │NTC      │    │蜂鸣器  │    │        │
          │TB6612 VCC│   │        │    │        │
          │电位器    │    │        │    │        │
          │LED(经电阻)│   │        │    │        │
          └─────────┘    └───────┘    └─────────┘
```

**关键原则：**
1. **所有GND必须连在一起**（STM32 GND ↔ 面包板GND ↔ 各模块GND）
2. **W25Q64必须用3.3V供电**，严禁接5V
3. 面包板电源模块需要单独供电（USB线或DC适配器接到模块的输入端）
4. STM32板可由ST-LINK USB或单独USB供电

### 4.4 物理安装建议

1. **面包板布局：** STM32最小系统板插在面包板一端，各模块围绕分布在周围
2. **电机固定：** 130电机可用双面胶固定在面包板边缘或桌面上
3. **风扇叶安装：** 将三叶风扇叶套在130电机轴上，用配套螺丝拧紧
4. **MPU6050固定：** 用双面胶将MPU6050模块贴在130电机的外壳侧面（金属壳效果最好）
5. **红外传感器定位：** 将反射式红外传感器对准风扇叶片，距离约1cm，略偏一个角度避免直接对射
6. **走线：** 电源线和信号线尽量分开走，减少干扰

## 五、STM32CubeMX配置指南

以下步骤将指导你在STM32CubeMX中完成所有外设配置。完成后直接生成CMake工程。

### 5.1 创建新工程

1. 打开STM32CubeMX
2. 点击 **File → New Project**
3. 在MCU选择器中搜索：`STM32F103C8`
4. 选择 **STM32F103C8T6**（LQFP48封装）
5. 点击 **Start Project**

### 5.2 SYS配置（调试接口+HAL时基）

路径：**System Core → SYS**

| 配置项 | 值 | 说明 |
|--------|-----|------|
| Debug | **Serial Wire** | 使用SWD调试，释放PB3/PB4供GPIO使用 |
| Timebase Source | **TIM1** | FreeRTOS占用SysTick，HAL时基改用TIM1 |

> **为什么用Serial Wire？** 默认的JTAG模式占用PA15、PB3、PB4三个引脚。改为Serial Wire只占用PA13、PA14，释放PB3(蜂鸣器)和PB4(可用)。

### 5.3 RCC时钟源配置

路径：**System Core → RCC**

| 配置项 | 值 |
|--------|-----|
| High Speed Clock (HSE) | **Crystal/Ceramic Resonator** |
| Low Speed Clock (LSE) | Disable |

### 5.4 时钟树配置

切换到 **Clock Configuration** 页签，按以下步骤配置：

1. **HSE输入频率：** 在 `Input frequency` 框中填入 **8** （MHz）——Blue Pill板载8MHz晶振
2. **PLL Source Mux：** 选择 **HSE**
3. **PLL Mul：** 选择 **×9**
4. **System Clock Mux：** 选择 **PLLCLK**
5. **HCLK (MHz)：** 输入 **72** → 回车，CubeMX会自动配置分频系数

配置完成后应看到：

```
HSE (8MHz) → PLL (×9) → SYSCLK (72MHz)
                        → AHB  /1 → HCLK  72MHz
                        → APB1 /2 → PCLK1 36MHz (Timer Clock 72MHz)
                        → APB2 /1 → PCLK2 72MHz (Timer Clock 72MHz)
```

> **注意：** 如果你的Blue Pill板载晶振不是8MHz（有些板子是12MHz或16MHz），需要修改HSE输入频率。查看板子上银色圆柱体元件上的标注。

### 5.5 USART1配置（串口调试/上报）

路径：**Connectivity → USART1**

| 配置项 | 值 |
|--------|-----|
| Mode | **Asynchronous** |
| Baud Rate | **115200** |
| Word Length | **8 Bits** |
| Stop Bits | **1** |
| Parity | **None** |
| Data Direction | **Receive and Transmit** |

引脚确认：PA9=USART1_TX, PA10=USART1_RX

> **中断：** 暂不开启USART中断，使用轮询方式收发。后续如果需要接收PC命令再开启。

### 5.6 I2C1配置（MPU6050 + OLED）

路径：**Connectivity → I2C1**

| 配置项 | 值 |
|--------|-----|
| Mode | **I2C** |
| I2C Speed Mode | **Standard Mode** |
| I2C Clock Speed | **100000** (100kHz) |

引脚确认：PB6=I2C1_SCL, PB7=I2C1_SDA

> **为什么用100kHz而不是400kHz？** 面包板上 jumper wire 较长，走线电容较大，100kHz更稳定。后续验证OK后可尝试400kHz提升速度。

### 5.7 SPI1配置（W25Q64）

路径：**Connectivity → SPI1**

| 配置项 | 值 | 说明 |
|--------|-----|------|
| Mode | **Full-Duplex Master** | 同时收发 |
| Frame Format | **Motorola** | |
| Data Size | **8 Bits** | |
| First Bit | **MSB First** | |
| Prescaler (Baud Rate) | **8** | APB2=72MHz/8=9MHz |
| CPOL | **Low** | 时钟空闲低电平 |
| CPHA | **1 Edge** | 第一个边沿采样 |
| CRC Calculation | **Disabled** | |
| NSS Signal Type | **Software** | 使用GPIO控制CS |

引脚确认：PA5=SPI1_SCK, PA6=SPI1_MISO, PA7=SPI1_MOSI

> CS引脚PA4需要在GPIO中手动配置为输出（见5.10节）。

### 5.8 ADC1配置（NTC温度 + 电位器电流）

路径：**Analog → ADC1**

**IN8设置（PB0 - NTC温度）：**

| 配置项 | 值 |
|--------|-----|
| IN8 | ✓ (勾选) |

**IN9设置（PB1 - 电位器电流）：**

| 配置项 | 值 |
|--------|-----|
| IN9 | ✓ (勾选) |

**ADC通用参数：**

| 配置项 | 值 | 说明 |
|--------|-----|------|
| Clock Prescaler | **PCLK2 div 6** | 72/6=12MHz（F103 ADC最高14MHz） |
| Resolution | **12 bits** | F103固定12位 |
| Scan Conversion Mode | **Enabled** | 扫描多通道（需先将Number of Conversion设为2，此选项才会解锁） |
| Continuous Conversion Mode | **Disabled** | 单次触发转换 |
| Discontinuous Conversion Mode | **Disabled** | |
| Number of Conversion | **2** | 2个通道 |
| Rank 1 - Channel | **Channel 8** | PB0, NTC |
| Rank 1 - Sampling Time | **55.5 Cycles** | |
| Rank 2 - Channel | **Channel 9** | PB1, 电位器 |
| Rank 2 - Sampling Time | **55.5 Cycles** | |

### 5.9 TIM2配置（电机PWM）

路径：**Timers → TIM2**

| 配置项 | 值 | 说明 |
|--------|-----|------|
| Channel1 | **PWM Generation CH1** | PA0输出PWM |
| Counter Settings - Prescaler | **71** | 72MHz/(71+1)=1MHz计数频率 |
| Counter Settings - Counter Mode | **Up** | |
| Counter Settings - Counter Period | **99** | 1MHz/(99+1)=10kHz PWM频率 |
| Counter Settings - auto-reload preload | **Enable** | |
| PWM Generation Channel 1 - Mode | **PWM mode 1** | |
| PWM Generation Channel 1 - Pulse | **50** | 默认50%占空比 |
| PWM Generation Channel 1 - Output compare preload | **Enable** | |

> **PWM频率说明：** 10kHz高于人耳可听频率范围，电机驱动不会产生啸叫。占空比范围0~99对应0%~100%。

### 5.10 GPIO配置

路径：**System Core → GPIO**

在Pinout视图中点击对应引脚，设置如下：

**输出引脚：**

| 引脚 | GPIO模式 | 标签(Label) | 初始电平 | 上拉/下拉 | 速度 |
|------|---------|-------------|---------|----------|------|
| PA4 | GPIO_Output | SPI1_CS | High | No pull | High |
| PA8 | GPIO_Output | LED_GREEN | Low | No pull | Low |
| PA11 | GPIO_Output | LED_YELLOW | Low | No pull | Low |
| PA12 | GPIO_Output | LED_RED | Low | No pull | Low |
| PB3 | GPIO_Output | BUZZER | High | No pull | Low |
| PB8 | GPIO_Output | MOTOR_AIN1 | Low | No pull | Low |
| PB9 | GPIO_Output | MOTOR_AIN2 | Low | No pull | Low |

> **说明：** PA4(W25Q64 CS)初始为High(未选中)。PB3(蜂鸣器)初始为High(不响，低电平触发)。LED初始为Low(不亮)。

**输入引脚：**

| 引脚 | GPIO模式 | 标签(Label) | 上拉/下拉 |
|------|---------|-------------|----------|
| PB5 | GPIO_EXTI5 | IR_SPEED | Pull-up |

**EXTI5配置（PB5红外测速）：**

路径：**System Core → GPIO → 在GPIO列表中找到PB5**

| 配置项 | 值 |
|--------|-----|
| GPIO mode | **External Interrupt Mode with Falling edge trigger detection** |
| GPIO Pull-up/Pull-down | **Pull-up** |

> 为什么选Falling edge？反射式红外传感器检测到反射物时DO输出低电平。叶片经过时产生下降沿。

### 5.11 NVIC中断优先级配置

路径：**System Core → NVIC**

| 中断源 | 使能 | 抢占优先级 | 子优先级 | 说明 |
|--------|------|-----------|---------|------|
| Time base: TIM1 | ✓ | 0 | 0 | HAL时基，最高优先 |
| EXTI line5 interrupt | ✓ | 6 | 0 | 红外测速脉冲 |
| TIM2 global interrupt | ✗ | — | — | PWM不需要中断 |
| USART1 global interrupt | ✗ | — | — | 暂用轮询模式 |

> **优先级规则：** FreeRTOS可管理的中断优先级数值必须 ≥ configLIBRARY_MAX_SYSCALL_INTERRUPT_PRIORITY（默认5）。所以EXTI设为6，可以被FreeRTOS管理。

| MAX_TASK_NAME_LEN | **8** | |
| IDLE_SHOULD_YIELD | **Enabled** | |
| USE_MUTEXES | **Enabled** | |
| USE_COUNTING_SEMAPHORES | **Enabled** | |
| USE_EVENT_GROUPS | **Enabled** | |
| USE_TIMERS | **Enabled** | |

#### 5.12.2 Tasks and Queues（通过CubeMX GUI添加）

在 **Tasks and Queues** 页签中添加以下6个任务：

**任务表：**

| 任务名 | 函数名 | 优先级 | 栈大小(words) |
|--------|--------|--------|--------------|
| SensorSample | StartTaskSensor | **osPriorityHigh** (6) | 256 |
| Control | StartTaskControl | **osPriorityAboveNormal** (5) | 128 |
| FeatureExtract | StartTaskFeature | **osPriorityNormal** (4) | 256 |
| Diagnosis | StartTaskDiagnosis | **osPriorityBelowNormal** (3) | 128 |
| Report | StartTaskReport | **osPriorityLow** (2) | 256 |
| Monitor | StartTaskMonitor | **osPriorityIdle** (1) | 128 |

> **优先级说明：** osPriorityHigh=6, osPriorityAboveNormal=5, osPriorityNormal=4, osPriorityBelowNormal=3, osPriorityLow=2, osPriorityIdle=1（CMSIS_V1定义值，FreeRTOS MAX_PRIORITIES=7时成立）

**队列表：**

| 队列名 | 长度 | 元素大小 |
|--------|------|---------|
| QueueRawData | 8 | 4（传递指针，sizeof(MotorRawData_t)=16） |
| QueueFeature | 4 | 4（传递指针，sizeof(MotorFeature_t)=32） |
| QueueDiagResult | 4 | 4（传递指针，sizeof(DiagResult_t)=24） |
| QueueControlCmd | 2 | 1（sizeof(uint8_t)） |

> **注意：** CMSIS_V1的`osMessagePut`只能传递`uint32_t`（4字节），因此前三个队列元素大小填4，实际传递的是结构体指针（`osMessagePut(queue, (uint32_t)&data, timeout)`）。QueueControlCmd传递uint8_t值本身，元素大小为1。结构体实际大小：MotorRawData_t=16字节，MotorFeature_t=32字节，DiagResult_t=24字节（含对齐填充）。

**互斥量：**

| 互斥量名 | 用途 |
|---------|------|
| MutexUart | 保护串口printf输出 |
| MutexI2C | 保护I2C总线（MPU6050和OLED共享）|

**事件组（CubeMX中不配置，生成代码后手动创建）：**

> CMSIS_V1没有事件组API，需使用FreeRTOS原生API。在生成代码后，在`freertos.c`的`MX_FREERTOS_Init()`函数中手动添加：
>
> ```c
> /* USER CODE BEGIN Variables */
> EventGroupHandle_t EventSystemHandle;
> /* USER CODE END Variables */
>
> /* 在 MX_FREERTOS_Init() 函数内，USER CODE BEGIN RTOS_EVENTS 处添加 */
> /* USER CODE BEGIN RTOS_EVENTS */
> EventSystemHandle = xEventGroupCreate();
> /* USER CODE END RTOS_EVENTS */
> ```
>
> 使用示例：
> - `xEventGroupSetBits(EventSystemHandle, 0x01)` — 设置bit0（传感器OK）
> - `xEventGroupWaitBits(EventSystemHandle, 0x01, pdFALSE, pdFALSE, portMAX_DELAY)` — 等待bit0
>
> 事件位定义：bit0=传感器OK, bit1=报警中, bit2=通信正常

#### 8.1.2 Task_FeatureExtract（特征提取任务）

```c
#define VIB_WINDOW_SIZE 64

void StartTaskFeature(void const *argument)
{
    MotorRawData_t raw;
    MotorFeature_t feature;
    float acc_buf[VIB_WINDOW_SIZE];  /* 合成加速度窗口 */
    int buf_idx = 0;
    int sample_count = 0;

    for (;;) {
        osEvent evt = osMessageGet(QueueRawDataHandle, osWaitForever);
        if (evt.status == osEventMessage) {
            MotorRawData_t *p = (MotorRawData_t *)evt.value.v;

            /* 计算合成加速度: a = sqrt(x^2 + y^2 + z^2) */
            float ax = p->acc_x / 16384.0f;  /* ±2g量程, 16384 LSB/g */
            float ay = p->acc_y / 16384.0f;
            float az = p->acc_z / 16384.0f;
            float acc_mag = sqrtf(ax*ax + ay*ay + az*az);

            acc_buf[buf_idx] = acc_mag;
            buf_idx = (buf_idx + 1) % VIB_WINDOW_SIZE;
            sample_count++;

            /* 窗口满后计算特征 */
            if (sample_count >= VIB_WINDOW_SIZE) {
                feature.timestamp = osKernelSysTick();
                feature.vib_rms   = calc_rms(acc_buf, VIB_WINDOW_SIZE);
                feature.vib_peak  = calc_peak(acc_buf, VIB_WINDOW_SIZE);
                feature.vib_pp    = calc_pp(acc_buf, VIB_WINDOW_SIZE);
                feature.vib_var   = calc_var(acc_buf, VIB_WINDOW_SIZE);
                feature.temperature = ntc_to_temperature(p->ntc_adc);
                feature.current     = adc_to_current(p->cur_adc);
                feature.rpm         = p->rpm;

                osMessagePut(QueueFeatureHandle, (uint32_t)&feature, 10);
                sample_count = VIB_WINDOW_SIZE / 2;  /* 50%重叠 */
            }
        }
    }
}
```

#### 8.1.3 Task_Diagnosis（诊断任务）

```c
void StartTaskDiagnosis(void const *argument)
{
    MotorFeature_t feature;
    DiagResult_t diag;
    DiagThreshold_t thresh = DEFAULT_THRESHOLDS;

    for (;;) {
        osEvent evt = osMessageGet(QueueFeatureHandle, 500);
        if (evt.status == osEventMessage) {
            MotorFeature_t *p = (MotorFeature_t *)evt.value.v;

            /* 计算健康指数 */
            int hi = 100;
            int fault = FAULT_NORMAL;

            /* 振动评分 */
            if (p->vib_rms >= thresh.vib_alarm) {
                hi -= 40; fault = FAULT_ALARM;
            } else if (p->vib_rms >= thresh.vib_warning) {
                hi -= 20;
                if (fault < FAULT_WARNING) fault = FAULT_WARNING;
            }

            /* 温度评分 */
            if (p->temperature >= thresh.temp_alarm) {
                hi -= 30;
                if (fault < FAULT_ALARM) fault = FAULT_ALARM;
            } else if (p->temperature >= thresh.temp_warning) {
                hi -= 15;
                if (fault < FAULT_WARNING) fault = FAULT_WARNING;
            }

            /* 电流评分 */
            if (p->current >= thresh.cur_alarm) {
                hi -= 25;
                fault = FAULT_SHUTDOWN;
            } else if (p->current >= thresh.cur_warning) {
                hi -= 12;
                if (fault < FAULT_WARNING) fault = FAULT_WARNING;
            }

            /* 限幅 */
            if (hi < 0) hi = 0;
            if (hi < 40) fault = FAULT_SHUTDOWN;

            /* 填充诊断结果 */
            diag.timestamp    = p->timestamp;
            diag.fault_level  = fault;
            diag.health_index = (uint8_t)hi;
            diag.vib_rms      = p->vib_rms;
            diag.temperature  = p->temperature;
            diag.current      = p->current;
            diag.rpm          = p->rpm;

            /* 发送到控制和上报任务 */
            osMessagePut(QueueDiagResultHandle, (uint32_t)&diag, 10);
            osMessagePut(QueueControlCmdHandle, (uint32_t)fault, 10);
        }
    }
}

### 9.1 MPU6050驱动（I2C）

**关键寄存器：**

| 地址 | 名称 | 功能 |
|------|------|------|
| 0x6B | PWR_MGMT_1 | 电源管理，写0x00唤醒 |
| 0x19 | SMPLRT_DIV | 采样率分频，0x00=1kHz |
| 0x1A | CONFIG | DLPF配置，0x01=184Hz带宽 |
| 0x1C | ACCEL_CONFIG | 加速度量程，0x00=±2g |
| 0x3B~0x40 | ACCEL_XOUT_H~ACCEL_ZOUT_L | 三轴加速度数据（6字节） |

**初始化序列：**

```c
#define MPU6050_ADDR         (0x68 << 1)  /* CubeMX HAL使用8位地址 */
#define MPU6050_REG_PWR_MGMT  0x6B
#define MPU6050_REG_SMPLRT    0x19
#define MPU6050_REG_CONFIG    0x1A
#define MPU6050_REG_ACCEL_CFG 0x1C
#define MPU6050_REG_ACCEL_XH  0x3B

HAL_StatusTypeDef mpu6050_init(I2C_HandleTypeDef *hi2c)
{
    uint8_t data;

    /* 唤醒 */
    data = 0x00;
    HAL_I2C_Mem_Write(hi2c, MPU6050_ADDR, MPU6050_REG_PWR_MGMT,
                       1, &data, 1, 100);
    HAL_Delay(100);

    /* 采样率：1kHz */
    data = 0x00;
    HAL_I2C_Mem_Write(hi2c, MPU6050_ADDR, MPU6050_REG_SMPLRT,
                       1, &data, 1, 100);

    /* DLPF：184Hz带宽 */
    data = 0x01;
    HAL_I2C_Mem_Write(hi2c, MPU6050_ADDR, MPU6050_REG_CONFIG,
                       1, &data, 1, 100);

    /* 加速度量程：±2g */
    data = 0x00;
    HAL_I2C_Mem_Write(hi2c, MPU6050_ADDR, MPU6050_REG_ACCEL_CFG,
                       1, &data, 1, 100);

    return HAL_OK;
}
```

**读取加速度（6字节连续读）：**

```c
HAL_StatusTypeDef mpu6050_read_accel(I2C_HandleTypeDef *hi2c,
                                      int16_t *ax, int16_t *ay, int16_t *az)
{
    uint8_t buf[6];
    HAL_StatusTypeDef ret;

    ret = HAL_I2C_Mem_Read(hi2c, MPU6050_ADDR, MPU6050_REG_ACCEL_XH,
                            1, buf, 6, 100);
    if (ret == HAL_OK) {
        *ax = (int16_t)((buf[0] << 8) | buf[1]);
        *ay = (int16_t)((buf[2] << 8) | buf[3]);
        *az = (int16_t)((buf[4] << 8) | buf[5]);
    }
    return ret;
}
```

### 9.2 NTC温度传感器驱动（ADC）

NTC热敏电阻通过分压电路输出模拟电压，ADC读取后换算为温度。

**换算公式：**

```
1. 计算NTC电阻值:
   R_ntc = R_fixed × (4095 / ADC_value - 1)
   其中 R_fixed 通常为 10KΩ（模块上的串联电阻）

2. Steinhart-Hart简化公式:
   T_kelvin = 1 / (1/T0 + (1/Beta) × ln(R_ntc / R0))
   T0 = 298.15K (25°C)
   R0 = 10000 (10KΩ at 25°C)
   Beta = 3950 (常见NTC参数，查看你的模块规格)

3. 转换为摄氏度:
   T_celsius = T_kelvin - 273.15
```

**驱动实现：**

```c
#include <math.h>

#define NTC_R_FIXED    10000.0f   /* 串联固定电阻 10KΩ */
#define NTC_R0         10000.0f   /* NTC 25°C时电阻 10KΩ */
#define NTC_T0         298.15f    /* 25°C in Kelvin */
#define NTC_BETA       3950.0f    /* B值 */
#define ADC_MAX        4095.0f    /* 10位ADC最大值 */

float ntc_to_temperature(uint16_t adc_value)
{
    if (adc_value == 0) return -999.0f;  /* 防止除零 */

    /* 计算NTC电阻 */
    float r_ntc = NTC_R_FIXED * (ADC_MAX / (float)adc_value - 1.0f);
    if (r_ntc <= 0) return -999.0f;

    /* Steinhart-Hart换算 */
    float t_kelvin = 1.0f / (1.0f/NTC_T0 + (1.0f/NTC_BETA) * logf(r_ntc/NTC_R0));

    return t_kelvin - 273.15f;
}
```

> **校准提示：** 在室温下测量ADC值，对比实际温度计，微调 `NTC_BETA` 值以提高精度。

## 十、特征提取算法

### 10.1 窗口策略

```
采集周期: 10ms → 100Hz采样率
窗口大小: 64个样本 → 覆盖640ms
窗口重叠: 50% → 每320ms输出一组特征
```

### 10.2 特征计算函数

```c
#include <math.h>
#include <float.h>

/* RMS (均方根) — 反映整体振动强度 */
static float calc_rms(const float *buf, int n)
{
    float sum = 0.0f;
    for (int i = 0; i < n; i++)
        sum += buf[i] * buf[i];
    return sqrtf(sum / n);
}

/* 峰值 — 反映最大冲击 */
static float calc_peak(const float *buf, int n)
{
    float max_val = 0.0f;
    for (int i = 0; i < n; i++) {
        float val = fabsf(buf[i]);
        if (val > max_val) max_val = val;
    }
    return max_val;
}

/* 峰峰值 — 反映振动波动范围 */
static float calc_pp(const float *buf, int n)
{
    float min_val = FLT_MAX;
    float max_val = -FLT_MAX;
    for (int i = 0; i < n; i++) {
        if (buf[i] < min_val) min_val = buf[i];
        if (buf[i] > max_val) max_val = buf[i];
    }
    return max_val - min_val;
}

/* 方差 — 反映振动稳定性 */
static float calc_var(const float *buf, int n)
{
    float mean = 0.0f;
    for (int i = 0; i < n; i++) mean += buf[i];
    mean /= n;

    float sum = 0.0f;
    for (int i = 0; i < n; i++) {
        float d = buf[i] - mean;
        sum += d * d;
    }
    return sum / n;
}
```

### 10.3 合成加速度计算

```c
/* 从三轴原始值计算加速度合成幅值 */
float calc_acc_magnitude(int16_t raw_x, int16_t raw_y, int16_t raw_z)
{
    float ax = raw_x / 16384.0f;  /* ±2g灵敏度: 16384 LSB/g */
    float ay = raw_y / 16384.0f;
    float az = raw_z / 16384.0f;
    return sqrtf(ax*ax + ay*ay + az*az);
}
```

> **说明：** 在静止状态下，合成加速度约为1g（重力）。电机振动会导致这个值在1g附近波动。RMS特征能反映振动偏离1g的程度。

## 十一、诊断模型设计

### 11.1 健康指数公式

```
HI = 100 - S_vib - S_temp - S_current
```

| 惩罚项 | 条件 | 扣分 |
|--------|------|------|
| S_vib | vib_rms ≥ 报警阈值 | -40 |
| S_vib | vib_rms ≥ 预警阈值 | -20 |
| S_temp | temperature ≥ 报警阈值 | -30 |
| S_temp | temperature ≥ 预警阈值 | -15 |
| S_current | current ≥ 报警阈值 | -25（同时触发停机） |
| S_current | current ≥ 预警阈值 | -12 |

### 11.2 故障等级判断

| 等级 | 条件 | LED | 蜂鸣器 | 电机 |
|------|------|-----|--------|------|
| 0 正常 | HI ≥ 80，无超限 | 绿灯 | 不响 | 正常运行 |
| 1 预警 | 60 ≤ HI < 80 | 黄灯 | 不响 | 正常运行 |
| 2 报警 | 40 ≤ HI < 60 | 红灯 | 响 | 正常运行 |
| 3 停机 | HI < 40 | 红灯 | 响 | 制动刹车 |

### 11.3 默认阈值配置

```c
DiagThreshold_t thresholds = {
    .vib_warning  = 0.30f,    /* 振动RMS > 0.3g 预警 */
    .vib_alarm    = 0.60f,    /* 振动RMS > 0.6g 报警 */
    .temp_warning = 50.0f,    /* 温度 > 50°C 预警 */
    .temp_alarm   = 70.0f,    /* 温度 > 70°C 报警 */
    .cur_warning  = 1.50f,    /* 电流 > 1.5A 预警 */
    .cur_alarm    = 2.00f,    /* 电流 > 2.0A 停机 */
};
```

> **调整提示：** 这些阈值是初始值。实际应根据你的电机和传感器实际数据进行标定。Phase 7测试时通过观察正常/异常状态下的RMS值来调整阈值。

### 14.2 实时数据采集与显示脚本

```python
"""
motor_dashboard.py — 电机预测性维护上位机
用法: python motor_dashboard.py [串口号]
示例: python motor_dashboard.py COM3
"""

import serial
import json
import sys
import time
import csv
from datetime import datetime
from collections import deque
import matplotlib.pyplot as plt
from matplotlib.animation import FuncAnimation

# ============ 配置 ============
SERIAL_PORT = sys.argv[1] if len(sys.argv) > 1 else 'COM3'
BAUD_RATE = 115200
WINDOW = 200  # 显示最近200个数据点

# ============ 数据缓冲 ============
timestamps = deque(maxlen=WINDOW)
data_rpm = deque(maxlen=WINDOW)
data_temp = deque(maxlen=WINDOW)
data_cur = deque(maxlen=WINDOW)
data_vib = deque(maxlen=WINDOW)
data_hi = deque(maxlen=WINDOW)
data_fault = deque(maxlen=WINDOW)

# ============ 串口 ============
ser = serial.Serial(SERIAL_PORT, BAUD_RATE, timeout=1)
print(f"已连接 {SERIAL_PORT} @ {BAUD_RATE}")

# ============ CSV文件 ============
csv_filename = f"motor_log_{datetime.now().strftime('%Y%m%d_%H%M%S')}.csv"
csv_file = open(csv_filename, 'w', newline='')
csv_writer = csv.writer(csv_file)
csv_writer.writerow(['timestamp', 'rpm', 'temp', 'current',
                      'vib_rms', 'health_index', 'fault_level'])

# ============ 图表 ============
fig, axes = plt.subplots(3, 2, figsize=(14, 9))
fig.suptitle('Motor Predictive Maintenance Dashboard', fontsize=14)

labels = ['RPM', 'Temperature (°C)', 'Current (A)',
          'Vibration RMS (g)', 'Health Index', 'Fault Level']
data_keys = [data_rpm, data_temp, data_cur, data_vib, data_hi, data_fault]

fault_colors = {0: 'green', 1: 'orange', 2: 'red', 3: 'darkred'}

def update(frame):
    """每100ms更新一次"""
    while ser.in_waiting > 0:
        line = ser.readline().decode('utf-8', errors='ignore').strip()
        if not line.startswith('{'):
            print(line)  # 打印系统日志
            continue

        try:
            j = json.loads(line)
        except json.JSONDecodeError:
            continue

        now = time.time()
        timestamps.append(now)
        data_rpm.append(j.get('rpm', 0))
        data_temp.append(j.get('temp', 0))
        data_cur.append(j.get('cur', 0))
        data_vib.append(j.get('vib_rms', 0))
        data_hi.append(j.get('hi', 0))
        data_fault.append(j.get('fault', 0))

        # 写CSV
        csv_writer.writerow([now, j.get('rpm', 0), j.get('temp', 0),
                             j.get('cur', 0), j.get('vib_rms', 0),
                             j.get('hi', 0), j.get('fault', 0)])

    # 更新图表
    for i, ax in enumerate(axes.flat):
        ax.clear()
        if len(data_keys[i]) > 1:
            color = 'green'
            if i == 5 and len(data_fault) > 0:  # Fault Level
                fl = list(data_fault)[-1]
                color = fault_colors.get(fl, 'green')
            ax.plot(list(data_keys[i]), color=color, linewidth=1)
            if i == 4:  # Health Index
                ax.axhline(y=80, color='green', linestyle='--', alpha=0.5)
                ax.axhline(y=60, color='orange', linestyle='--', alpha=0.5)
                ax.axhline(y=40, color='red', linestyle='--', alpha=0.5)
        ax.set_title(labels[i], fontsize=10)
        ax.grid(True, alpha=0.3)

    fig.tight_layout(rect=[0, 0, 1, 0.96])

ani = FuncAnimation(fig, update, interval=100, cache_frame_data=False)
plt.show()

# 关闭
csv_file.close()
ser.close()
print(f"数据已保存到 {csv_filename}")
```

## 十六、迁移到STM32H750的升级路线

### 16.1 架构可迁移性

本Demo的软件架构设计时已考虑迁移兼容性：

| 模块 | F103 Demo | H750正式版 | 迁移难度 |
|------|-----------|-----------|---------|
| FreeRTOS任务框架 | 6任务+队列 | 直接复用，可扩展更多任务 | 低 |
| 数据结构 | motor_types.h | 完全复用 | 无 |
| 诊断模型 | 规则阈值 | 可扩展为ML模型 | 中 |
| 通信 | 串口JSON | 替换为LwIP+MQTT | 高（重写通信层） |
| 振动采集 | MPU6050 I2C 100Hz | 升级SPI传感器 3.2kHz | 中 |
| 特征提取 | 时域RMS/峰值 | 加入CMSIS-DSP FFT | 中 |
| 存储无 | W25Q64 SPI | 升级SD卡/W25Q128 | 低 |
| OLED | SSD1306 I2C | 升级更大屏幕 | 低 |

### 16.2 功能升级清单

迁移到STM32H750后应实现的功能：

1. **高速采样**：3.2kHz+振动采集，DMA传输
2. **FFT频域分析**：CMSIS-DSP 1024点FFT，提取主频和频域能量
3. **以太网通信**：LwIP协议栈 + MQTT上报
4. **Modbus TCP**：工业协议接入
5. **OTA升级**：远程固件更新
6. **离线缓存**：SD卡或大容量Flash存储历史数据
7. **Web配置页面**：嵌入式Web服务器
8. **更复杂诊断**：机器学习模型（可选）

### 16.3 迁移步骤建议

1. 将FreeRTOS任务框架和数据结构代码直接复制到H750工程
2. 替换HAL底层驱动（F1 HAL → H7 HAL）
3. 升级传感器驱动（如MPU6050 I2C → ICM42688 SPI）
4. 添加LwIP和MQTT中间件
5. 添加CMSIS-DSP FFT功能
6. 整合测试

## 附录A：常见问题排查

### A.1 烧录问题

| 现象 | 可能原因 | 解决方法 |
|------|---------|---------|
| ST-LINK连不上 | SWD线未接好 | 检查SWDIO/SWCLK/GND三线 |
| 烧录后不运行 | BOOT0引脚为高 | 确认BOOT0接GND或检查跳线帽 |
| 程序跑飞 | 时钟配置错误 | 确认HSE频率与板载晶振匹配 |

### A.2 串口问题

| 现象 | 可能原因 | 解决方法 |
|------|---------|---------|
| 串口无输出 | CH340驱动未安装 | 安装CH340驱动，检查设备管理器 |
| 乱码 | 波特率不匹配 | 确认两端都是115200 |
| TX/RX接反 | 线序错误 | TX接RX，RX接TX，交叉连接 |

### A.3 I2C问题

| 现象 | 可能原因 | 解决方法 |
|------|---------|---------|
| I2C始终TIMEOUT | 接线错误或无上拉 | 检查SCL/SDA，确认模块有上拉电阻 |
| 地址不对 | AD0引脚状态 | AD0=GND→0x68, AD0=VCC→0x69 |
| 数据全为0 | MPU6050未唤醒 | 确认写0x00到PWR_MGMT_1寄存器 |

### A.4 电机问题

| 现象 | 可能原因 | 解决方法 |
|------|---------|---------|
| 电机不转 | TB6612供电不足 | 确认VM=5V, VCC=3.3V, STBY=3.3V |
| 电机转速很低 | PWM占空比太小 | 增大占空比到50以上 |
| 电机抖动不转 | AIN1/AIN2同时为1 | 检查GPIO输出，不应同时为高 |

### A.5 FreeRTOS问题

| 现象 | 可能原因 | 解决方法 |
|------|---------|---------|
| 系统反复重启 | 任务栈溢出 | 增大对应任务的栈大小 |
| HardFault | 空指针或数组越界 | 使用调试器查看Fault寄存器 |
| free_heap持续下降 | 内存泄漏 | 检查是否有osMalloc未释放 |
| 某个任务不运行 | 优先级配置错误 | 检查任务优先级和队列阻塞 |

## 附录B：硬件模块照片参考说明

以下描述帮助你识别各模块的引脚（配合模块实物查看）：

**MPU6050模块（方形小板）：**
- 正面有银色金属芯片（MPU6050）
- 8个引脚排针在一侧
- 引脚丝印：VCC, GND, SCL, SDA, XDA, XCL, AD0, INT

**0.96寸OLED（长方形小板）：**
- 正面有黑色显示区域
- 4个引脚排针在底部（需要你自己焊接）
- 引脚丝印：VCC, GND, SCL, SDA

**NTC热敏电阻模块：**
- 板上有蓝色热敏电阻元件
- 板上有LM393比较器和灵敏度调节电位器
- 4引脚：VCC, GND, AO(模拟), DO(数字)

**反射式红外传感器模块：**
- 板上有黑色红外发射管和透明红外接收管并排
- 板上有LM393和灵敏度电位器
- 4引脚：VCC, GND, AO, DO

**TB6612模块（较大方形板）：**
- 正面有TB6612FNG芯片
- 16个引脚分两排
- 丝印标注：VM, VCC, GND, AO1, AO2, BO1, BO2, AIN1, AIN2, BIN1, BIN2, PWMA, PWMB, STBY

**W25Q64模块：**
- 正面有W25Q64芯片（8脚）
- 6个排针引脚
- 丝印：VCC, GND, CS, DO, DI, CLK（或类似标注）
